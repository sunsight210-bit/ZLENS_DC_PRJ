# 电动变倍镜头 STM32 项目需求报告

> **项目**：电动变倍镜头（直流电机驱动）
> **主控**：STM32F103RCT6
> **版本**：v1.0
> **日期**：2026-03-16

---

## 1. 项目概述

本项目开发一款电动变倍镜头控制系统，通过 STM32 接收 PC 串口命令，驱动直流减速电机实现镜头变倍。系统基于堵转电流检测实现限位保护，通过增量编码器实现精准位置控制，通过 FRAM 实现掉电位置记忆。

### 2.4 五线信号说明

| 信号 | 作用 | 详细说明 |
|------|------|---------|

### 2.5 电流检测电路

```
电机电流 → A4950 LSS(Pin7) → R10(0.2Ω) → GND
                                  |
                          R88(100Ω) → LM321A(U8) +IN
                                        | (增益=1, R5未贴)
                                   U8 OUT → R89(100Ω)
                                        |
                                   C8(1μF) 对地滤波
                                        |
                                PC0 (ADC12_IN10) → STM32
```

- 采样电阻 R10 = 0.2Ω
- LM321A 配置为电压跟随器（R5 未贴片，增益=1）
- R2 = 10kΩ（输出到反相输入反馈）
- 低通滤波：R89(100Ω) + C8(1μF)，截止频率 ≈ 1.59 kHz

### 2.6 电源检测电路

```
12V (VCC_IN) → R99(30kΩ) → PB0(ADC12_IN8) → R83(10kΩ) → GND
```

分压比 1:4，12V 输入对应 PB0 = 3.0V。

### 2.7 电流与 ADC 对应关系

| 电机状态 | 电流 | R10 压降 | PC0 电压 | ADC 值 (12-bit) |
|----------|------|----------|----------|-----------------|
| 空载/轻载 | ~50 mA | 10 mV | 10 mV | ~12 |
| 额定运行 | ~200 mA | 40 mV | 40 mV | ~50 |
| 高负载 | ~500 mA | 100 mV | 100 mV | ~124 |
| 堵转（参考值） | ~1297 mA | 259 mV | 259 mV | ~321 |
| 堵转（最大值） | ~1480 mA | 296 mV | 296 mV | ~367 |

### 2.8 PWM 驱动方式

电机驱动采用 **TIM3 硬件 PWM**，PA6 映射到 TIM3_CH1，PA7 映射到 TIM3_CH2：

- TIM3 配置为 PWM 模式 1，向上计数
- PWM 频率默认 15 kHz，可调范围 10~30 kHz
- 64 MHz / 15 kHz ≈ 4267，ARR = 4266，占空比分辨率约 12 位
- 硬件 PWM 输出稳定，无 CPU 中断开销

A4950 控制真值表：

| 动作 | PA6 (IN1) | PA7 (IN2) | 说明 |
|------|-----------|-----------|------|
| 正转（倍率增大） | PWM | 0 | 占空比控制速度 |
| 反转（倍率减小） | 0 | PWM | 占空比控制速度 |
| 刹车 | 1 | 1 | 慢衰减，快速制动 |
| 自由停止 | 0 | 0 | 高阻态，惯性滑行 |

DAC 电流限制：**I_TripMAX = V_REF / (10 × R_S) = PA5 电压 / 2**

## 3. 通信协议（V2.5）

### 3.1 串口参数

波特率 115200，8 数据位，1 停止位，无校验，无流控。

### 3.2 帧格式（固定 6 字节）

| 帧头 | 命令字 | 参数高字节 | 参数低字节 | CRC16 低字节 | CRC16 高字节 |
|------|--------|-----------|-----------|-------------|-------------|
| 0xA5 | CMD | PARAM_H | PARAM_L | CRC_L | CRC_H |

CRC16/MODBUS 算法，初始值 0xFFFF，多项式 0xA001，计算范围 buf[0]~buf[3]。

### 3.3 命令列表

| 命令字 | 名称 | 参数含义 |
|--------|------|---------|
| 0x01 | 归零 | 保留 |
| 0x02 | 强制停止 | 保留 |
| 0x03 | 查询堵转计数 | 保留 |
| 0x10 | 设置倍率 | 倍率×10，仅限映射表中的 14 个标定值（6/8/10/12/15/18/20/25/30/35/40/50/60/70） |
| 0x11 | 增加倍率 | 增量×10 |
| 0x12 | 减少倍率 | 减量×10 |
| 0x20 | 查询当前倍率 | 保留 |
| 0x21 | 查询运行状态 | 保留 |
| 0x22 | 查询速度 | 保留 |
| 0x23 | 查询镜头类型 | 保留 |
| 0x24 | 查询倍率范围 | 保留 |
| 0x30 | 循环变倍 | 步距+停留时间 |
| 0x31 | 停止循环 | 保留 |

### 3.4 应答机制

- **握手应答**：收到合法帧后原样回传 6 字节
- **执行反馈**：动作完成后发送新帧（重新计算 CRC）
- **忙碌拒绝**：运动中收到新变倍命令，返回 CMD=0x03, PARAM=0x000E

## 4. 编码器计数体系

### 4.1 4 倍频原理

TIM8 设为编码器模式（TI1 and TI2），A/B 通道的每个上升沿和下降沿都计数：

```
1 个 A/B 脉冲周期 → 4 个计数（A↓ +1, B↓ +1, A↑ +1, B↑ +1）
```

### 4.2 计数换算

| 级别 | 每转计数 |
|------|---------|
| 电机轴 1 转 | 512 × 4 = **2,048 计数** |
| 输出轴 1 转 | 2,048 × 104.77 = **214,569 计数** |

### 4.3 TIM8 溢出处理

TIM8 为 16 位计数器（0~65535），约 32 圈电机轴转即溢出，远不够表达全行程。需软件扩展：

```c
volatile int32_t encoder_overflow = 0;

// TIM8 溢出中断（优先级最高）
void TIM8_UP_IRQHandler(void) {
    if (正转溢出) encoder_overflow++;
    if (反转下溢) encoder_overflow--;
}

int32_t get_position(void) {
    return encoder_overflow * 65536 + (uint16_t)TIM8->CNT;
}
```

## 6. 核心功能需求

### 6.1 需求 1：堵转限位识别

#### 6.1.1 需求描述

电机撞到机械极限时发生堵转，ADC 检测到电流突增。根据**当前运动方向**判定是正限位还是负限位：正转堵转 = 正限位，反转堵转 = 负限位。编码器无脉冲变化作为辅助确认。

#### 6.1.2 检测信号

- **主判据**：PC0 (LSS_BACK) ADC 值持续超过堵转阈值
- **辅助判据**：编码器无脉冲变化（电机轴不转）
- **方向判据**：当前电机运动方向标志

#### 6.1.3 堵转阈值

| 方案 | 阈值 | 说明 |
|------|------|------|
| 固定阈值 | ADC > 200（约 650 mA） | 简单可靠 |
| 动态标定 | baseline_current × 3 | 首次归零时采样正常电流，乘以 3 |

建议两者结合：动态标定值优先，固定值作为保底。

#### 6.1.4 确认策略

| 防护机制 | 参数 | 说明 |
|----------|------|------|
| 启动屏蔽窗口 | 200 ms | 电机启动瞬间大电流，不判断堵转 |
| 连续采样确认 | 1000 次（1 秒） | ADC 连续 1 秒超阈值才确认堵转 |
| 编码器辅助 | 500 ms 无脉冲 | PWM 有输出但编码器无变化 |

#### 6.1.5 限位方向判定

```
堵转确认
  |
  ├── 当前方向 = 正转 → 正限位 (LIMIT_MAX) → 记录 max_position
  └── 当前方向 = 反转 → 负限位 (LIMIT_MIN) → 记录 min_position
```

#### 6.1.6 堵转保护流程

```
堵转确认 → 立即刹车(IN1=IN2=1)
         → 根据方向标记限位类型和位置
         → stall_count++ → 存入 FRAM
         → stall_count >= 5 → 发送 0xE1 报警，进入故障状态
```

#### 6.1.7 过流保护（独立于堵转，快速响应）

ADC > 350（约 1400 mA）连续 10 ms → 立即刹车 + DAC 输出 0V → 发送 0xE2 → 进入故障状态。

### 6.2 需求 2：软限位零点

#### 6.2.1 需求描述

在物理限位（堵转点）往中心方向退回 **50 个编码器脉冲**（4 倍频 = 200 个计数）处设置软限位，作为实际运动边界。零点可配置挂靠在正限位侧或负限位侧。避免正常运动中触发堵转，保护齿轮和电机。

#### 6.2.2 软限位定义

```
物理负限位      软负限位(0.6x)                         软正限位(7.0x)      物理正限位
    |  ←50脉冲→   |                                      |  ←50脉冲→   |
    |  ←200计数→   |<========= 有效行程 ==========>|   ←200计数→  |
    0             200                             range-200          range
```

- 物理负限位 = 编码器计数 0（首次标定时设定）
- 软负限位 = 200 计数（50 脉冲 × 4 倍频）
- 软正限位 = range - 200 计数
- 物理正限位 = range 计数
- **有效行程 = range - 400 计数**

#### 6.2.3 倍率映射（在有效行程内）

位置控制基于映射表直接查表，不使用连续公式。14 个倍率各自对应固定的编码器计数值（见 5.1 节映射表），目标位置 = 查表计数 + soft_limit_offset。

#### 6.2.4 零点配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| soft_limit_offset | 200 计数 | 50 脉冲 × 4 倍频，可调 |
| zero_reference | 0 或 1 | 0=零点挂靠负限位侧，1=挂靠正限位侧 |

零点挂靠说明：
- **挂靠负限位侧**（默认）：0.6x 位置 = 软负限位，向正方向运动增大倍率
- **挂靠正限位侧**：7.0x 位置 = 软正限位，向负方向运动减小倍率

#### 6.2.5 软限位保护

正常运动中，位置控制在软限位范围内，不会触及物理限位：

```
目标位置检查:
  if (target < soft_min) target = soft_min;
  if (target > soft_max) target = soft_max;

运动中位置监测:
  if (position <= soft_min && direction == REVERSE) → 停止
  if (position >= soft_max && direction == FORWARD) → 停止
```

#### 6.2.6 首次全行程标定流程

```
(1) 反转低速 → 堵转 → 物理负限位
    TIM8->CNT = 0, overflow = 0         ← 此处定义为计数 0

(2) 刹车 → 延时 200ms → 退回

(3) 低速正转 → 找 Z 脉冲 → 记录 z_offset

(4) 继续正转 → 堵转 → 物理正限位
    记录 range = get_position()          ← 如 206,821

(5) 计算:
    soft_min = 200                       ← 50 脉冲 × 4
    soft_max = range - 200               ← 如 206,621
    effective_range = range - 400         ← 如 206,421

(6) 退回到 soft_min → 标定完成

(7) 保存到 FRAM: range, soft_min, soft_max, z_offset, 映射表参数
```

### 6.3 需求 3：掉电免归零

#### 6.3.1 需求描述

运动中或静止时掉电，通过 FRAM 实时保存编码器位置。重新上电后直接从 FRAM 恢复位置，无需回零即可接受变倍命令并精准到达指定倍率。可选用 Z 脉冲做快速校验，确认位置无偏差。

#### 6.3.2 位置保存策略

| 保存时机 | 保存内容 | 说明 |
|----------|---------|------|
| 变倍到位 | position, zoom_x10 | 每次运动完成后保存 |
| 运动过程中 | position | 每 500ms 定期保存 |
| 掉电检测触发 | position | 紧急保存，<1ms 完成 |
| 堵转事件 | position, stall_count, 限位位置 | 故障记录 |
| 归零完成 | 全部标定参数 | 完整保存 |

#### 6.3.3 掉电检测

```
PB0 (POWER_BACK) ADC 每 1ms 采样:
  12V 正常 → ADC ≈ 3723
  10V 阈值 → ADC ≈ 3103

  ADC < 3103 连续 5ms → 确认掉电 → 触发紧急保存
```

#### 6.3.4 紧急保存流程（< 1ms 完成）

```
掉电确认
  |
(1) 刹车 (IN1=IN2=1，利用反电动势制动，不依赖外部电源)
(2) 读取 TIM8 当前位置
(3) FRAM: 解除 WP → 写入 position + position_valid=0xFF → 恢复 WP
    (FRAM 写入 58 字节 @SPI 10MHz ≈ 50μs)
```

104.77:1 高减速比确保刹车后输出轴几乎瞬间停止，位置误差极小。

#### 6.3.5 上电恢复流程

```
上电 → 初始化外设
  |
读取 FRAM → CRC 校验
  ├── 失败 → 数据无效 → 必须完整归零
  └── 成功 → 检查标志
       ├── homing_done == 0 → 从未归零 → 必须完整归零
       ├── position_valid != 0xFF → 位置不可信 → 必须完整归零
       └── 全部有效 → 快速恢复
            |
            (1) 从 FRAM 加载位置到编码器计数器:
                TIM8->CNT = saved_position % 65536
                encoder_overflow = saved_position / 65536
            |
            (2) 加载 range, soft_min, soft_max, z_offset 等参数
            |
            (3) 可选: Z 脉冲快速验证
                低速运动找最近 Z 脉冲（最多 2048 计数）
                比较期望 Z 位置 vs 实际 Z 位置
                ├── 偏差 < 100 计数 → 位置有效
                └── 偏差 > 100 计数 → 位置偏移，完整归零
            |
            (4) 系统状态 → READY
                可立即接受变倍命令
```

#### 6.3.6 各场景分析

| 掉电场景 | FRAM 位置精度 | 恢复后行为 |
|----------|-------------|-----------|
| 电机静止时掉电 | 精确 | 直接恢复，无偏差 |
| 电机运动中掉电（检测到） | 极小偏差 | 恢复 + 可选 Z 验证 |
| 电机运动中掉电（未检测到） | 最多 500ms 运动偏差 | 恢复 + 建议 Z 验证 |
| 从未归零就掉电 | 无效 | 必须完整归零 |

#### 6.3.7 精准到达保障

掉电恢复后，精准到达指定倍率的关键链路：

```
上位机: 设置倍率 2.0x
  ↓
查映射表: 2.0x → 目标计数 89,404
  ↓
当前位置: 从 FRAM 恢复的值，如 28,000
  ↓
计算: 需正转 89,404 - 28,000 = 61,404 计数
  ↓
启动电机 → 加速 → 匀速 → 减速 → 到位(±50 计数死区)
  ↓
反馈: CMD=0x10 当前倍率 + CMD=0x02 到位通知
```

## 8. 系统工作流程总览

### 8.1 首次上电（全行程标定）

```
上电 → 外设初始化 → FRAM 读取 → magic_number 无效
  |
全行程标定:
  (1) 反转低速 → 堵转(负限位) → 设计数器=0
  (2) 退回 → 找 Z 脉冲 → 记录 z_offset
  (3) 正转 → 堵转(正限位) → 记录 range
  (4) 计算 soft_min/soft_max
  (5) 运动到 soft_min(0.6x)
  (6) 全部参数存入 FRAM
  (7) 系统就绪 → 通知上位机
```

### 8.2 正常上电（FRAM 有效，免归零）

```
上电 → 外设初始化 → FRAM 读取 → CRC 通过 + homing_done + position_valid
  |
快速恢复:
  (1) 编码器计数器 = FRAM 保存位置
  (2) 可选 Z 脉冲验证
  (3) 系统就绪 → 接受变倍命令
```

### 8.3 正常变倍

```
收到设置倍率命令
  → 握手应答
  → 查映射表得目标计数
  → 计算方向和距离
  → 加速 → 匀速 → 减速 → 到位
  → 刹车 → 保存位置到 FRAM
  → 反馈当前倍率 + 到位通知
```

### 8.4 堵转保护

```
运动中 ADC 电流突增
  → 连续 1 秒确认（编码器辅助）
  → 刹车 → 根据方向标记限位
  → stall_count++ → FRAM 保存
  → 达到 5 次 → 0xE1 报警
```

### 8.5 掉电保护

```
PB0 检测电压下降
  → 连续 5ms 确认
  → 刹车(反电动势制动) → 读位置 → FRAM 紧急保存
  → 断电（位置已保存）
```

## 9. 接线自检功能

上电自检时可检测硬件接线错误：

| 自检项 | 方法 | 判定 |
|--------|------|------|
| 编码器方向 | 短暂驱动正转，检查编码器计数方向 | 计数递增=正确，递减=接反 |
| 编码器连接 | 驱动电机，检查计数是否变化 | 无变化=未接或损坏 |
| Z 通道 | 归零过程中超时未检测到 Z 脉冲 | 超时=Z 通道故障 |
| ADC 电流 | 电机静止时 ADC 应接近 0 | 异常偏高=电路故障 |
| 电源电压 | PB0 ADC 应在合理范围 | 过低=供电不足 |
