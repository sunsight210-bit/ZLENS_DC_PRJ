# 堵转电流测试报告

**日期：** 2026-03-21
**固件版本：** v1.0.0
**测试方法：** HOMING param=2 触发自动堵转测试序列

## 测试条件

- 供电电压：11.9V（ADC=2014）
- 空载基线电流：ADC=40（约 161mA，按旧公式）
- 电机减速比：104.77:1 + 外部齿轮 4:1 = 419.08:1
- 测试位置：从 2.0x（354,918 counts）向负极限堵转
- 每档堵转持续：4000ms，采样 4000 次（1ms/次）
- ADC 滤波：EMA α=1/16（ALPHA_SHIFT=4）
- PWM 频率：对应 PWM_ARR=4266

## 关键发现

### 1. ADC 波动极大（PWM 采样未同步）

所有测试的 ADC_min ≈ 20、ADC_max ≈ 4089（接近 12-bit 满幅）。
原因：ADC 采样与 PWM 不同步，PWM 导通时 ADC 读到高值，关断时读到近零值。
ADC 平均值 ≈ 占空比 × 峰值电流 ADC。

### 2. 峰值电流恒定（符合物理预期）

| 占空比 | ADC_avg | ADC_avg / 占空比 = 峰值 ADC |
|--------|---------|--------------------------|
| 30% | 940 | **3133** |
| 50% | 1720 | **3440** |
| 80% | 2720 | **3400** |

峰值 ADC 约 **3300~3440**，说明堵转时峰值电流恒定（由电机绕组电阻和供电电压决定），与占空比无关。

### 3. 电流校准

外部仪器在 80% 堵转时测得 **800mA**（均值）。
- 峰值电流 = 800mA / 0.8 = **1000mA (1.0A)**
- 峰值 ADC ≈ 3400
- 校准系数：**1 ADC ≈ 0.294 mA**（峰值），或 **ADC_avg × 0.294 ≈ 均值电流 mA**

旧公式 `ADC × 16500 / 4095`（假设 Rsense=0.2Ω 无放大）给出值偏大约 **13.7 倍**，
说明电流检测回路存在约 13.7 倍的增益（可能是 Rsense + 运放增益的组合）。

### 4. 正常运行速度下的堵转预估

正常 MAX_SPEED = 2400（56% 占空比），堵转 ADC_avg 推算：
- 线性插值：1720 + (2720 - 1720) × (56 - 50) / (80 - 50) = **1920**
- 对应均值电流 ≈ 1920 × 0.294 ≈ **565mA**

## 旧保护阈值问题

| 参数 | 旧值 | 对应电流 | 问题 |
|------|------|---------|------|
| `STALL_THRESHOLD` | 200 | ~59mA | **过低**：正常带载运行就可能超过，导致误触发 |
| `OVERCURRENT_THRESHOLD` | 1200 | ~353mA | 50% 堵转均值(1720)就超过，无法在堵转前保护 |
| `STALL_CONFIRM_COUNT` | 1000 | 1 秒 | 合理 |
| `OVERCURRENT_CONFIRM` | 100 | 100ms | 可加快 |

## 最终参数选择

| 参数 | 新值 | 对应电流 | 依据 |
|------|------|---------|------|
| `STALL_THRESHOLD` | **600** | ~176mA | 空载基线 40 → 带载运行估计 100~300 → 600 留 2 倍裕度；56% 堵转 ~1920 远高于此，确保可靠检测 |
| `OVERCURRENT_THRESHOLD` | **3000** | ~882mA | 80% 堵转均值 2720 → 3000 留 10% 裕度；对应峰值 ADC ~3750 ≈ 1.1A，保护驱动芯片 |
| `STALL_CONFIRM_COUNT` | **1000** | 1 秒 | 不变，避免加速/减速瞬态误触发 |
| `OVERCURRENT_CONFIRM` | **50** | 50ms | 从 100ms 缩短到 50ms，更快保护驱动芯片免受持续大电流损伤 |

### 保护逻辑层次

```
正常运行 ADC_filtered ≈ 40~300
         ↓ 超过 600 且持续 1s
堵转检测 → emergency_stop + 报告 STALL_STOP(0xE1)
         ↓ 超过 3000 且持续 50ms
过流保护 → emergency_stop + 报告 OVERCURRENT(0xE2)
```
