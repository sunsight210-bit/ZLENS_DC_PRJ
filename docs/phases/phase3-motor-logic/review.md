## 经验教训

1. **编码器极性必须早期验证** — A/B 通道接反导致电机方向与编码器计数相反，运动控制完全失效。应在第一次上板时就做方向验证（Test 0），而不是等到归零流程出问题才发现。

2. **所有电机运动循环必须有超时保护** — wait-for-IDLE 循环无超时 + 编码器方向错误 = 电机堵转在限位持续通电 → 电路板/电机过热。任何驱动电机的循环都必须有绝对超时 + 编码器停转检测双重保护。

3. **溢出方向判断用 DIR 位而非 CNT 值** — `TIM8->CNT < 32768` 在 STM32F103 上不可靠（寄存器高 16 位可能非零），CR1.DIR 位是硬件只读位，更可靠。

4. **分压电阻变更需同步更新软件** — 硬件 R_top/R_bottom 变更后，ADC 转换公式和阈值必须同步修改，否则失压保护失效。

5. **实际编码器行程远超预估** — 预估 28,000 counts，实测 346,292 counts。后续设计不应依赖预估行程，应以归零校准的实测值为准。
