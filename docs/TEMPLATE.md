# 阶段文档模板

创建新阶段时，复制对应模板内容到 `docs/phases/phaseN-xxx/` 目录下。

---

## plan.md

```markdown
# Phase N: 阶段名称

## 目标
1-2 句描述本阶段要达成什么。

## 设计来源
引用 spec.md（如有），或简述设计决策背景。

## 改动文件清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `path/to/file.cpp` | 新增/修改 | 说明 |

## 设计要点
- 关键技术决策 1
- 关键技术决策 2

## 进度清单
- [ ] 任务 1
- [ ] 任务 2
- [ ] Mock 测试全部通过
- [ ] 固件编译通过
- [ ] 板上测试通过
- [ ] review.md 已填写
- [ ] git commit & push
```

---

## verify.md

```markdown
# Phase N 验证记录

## Mock 测试

| 模块 | 测试数 | 通过 | 覆盖率 | 日期 |
|------|--------|------|--------|------|
| | | | | |

## 固件编译

| 日期 | text | data | bss | 结果 |
|------|------|------|-----|------|
| | | | | |

## 板上测试

脚本: `scripts/phaseN_verify.py --port /dev/ttyUSB0`

| 测试项 | 结果 | 日期 | 备注 |
|--------|------|------|------|
| | | | |
```

---

## review.md

```markdown
# Phase N Review

## 代码审查
- [ ] 命名规范符合 coding-style.md
- [ ] TDD 覆盖率 >80%
- [ ] 产品代码与测试代码边界清晰

## 设计审查
（从 plan.md 的"设计要点"逐项确认是否合理实现）
- [ ] 设计要点 1
- [ ] 设计要点 2

## 改动文件清单
（列出本阶段所有新增/修改的文件）

## 待改进
（完成后填写）
```

---

## spec.md（可选，复杂阶段使用）

由 brainstorming skill 产出的设计规格文档。简单阶段可跳过，直接写 plan.md。
