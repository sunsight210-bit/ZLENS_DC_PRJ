# 编码器 Count 偏移 vs 图像像素精度测试

## 测试目的

确定编码器定位精度（DEADZONE）对图像测量精度的实际影响，为 DEADZONE 参数选择提供量化依据。

当前 PID 控制 DEADZONE=1（±1 count），到位精度高但末端 settling 时间较长。需要评估放宽 DEADZONE 是否会影响图像测量质量，找到精度与速度的平衡点。

## 测试环境

| 项目 | 值 |
|------|-----|
| 测试日期 | 2026-04-02 |
| 测试倍率 | 3.0X（编码器基准 count = 38539） |
| 编码器 | AS5311 磁编码器，1024 counts/pole pair，64 pole pairs，全程 65536 counts |
| 图像测量 | 圆目标直径（pixel），每个位置测量两次取值 |
| 诊断固件 | `Test/diag/zoom_precision_diag.cpp` |

## 测试方法

1. 每个测试点先执行完整回零（快回零 → 退回 → 慢回零 → HOME_OFFSET），消除间隙和反向误差
2. 从零位正向走到目标位置（单向运动，无反向）
3. 到位后手动测量图像中圆目标的直径（pixel）
4. 共 13 个测试点：3.0X 基准 + ±2/±5/±10/±20/±50/±100 count 偏移

## 分析

- 灵敏度约 **0.03 pixel/count**
- 两次测量本身波动 ~0.3 pixel（测量噪声）
- ±5 counts 内：Δpixel < 1 → 亚像素，不可分辨
- ±10 counts 内：Δpixel < 1.2 → 约 1 像素
- ±20 counts：Δpixel ≈ 1 pixel
- ±100 counts：Δpixel ≈ 3 pixel

## 结论

建议 DEADZONE 从 1 放宽到 **3**：
- ±3 counts ≈ 0.1 pixel 误差，完全在测量噪声（~0.3 pixel）之内
- 可显著减少 PID 末端 settling 时间
- 对图像测量精度无可见影响
