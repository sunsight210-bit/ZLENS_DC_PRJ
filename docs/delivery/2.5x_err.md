# 2.5X 重复定位精度测试 (DEADZONE=1)

## 测试条件

| 项目 | 值 |
|------|-----|
| 测试倍率 | 2.5X (zoom_x10=25) |
| 编码器目标 | 33169 counts |
| DEADZONE | 1 |
| 测量方式 | 图像中目标尺寸 (pixel) |

## 手动测量数据

### 从不同方向到达 2.5X

| 路径 | Pixel 测量值 |
|------|-------------|
| 0.6X → 2.5X | 700.852 |
| 4.0X → 2.5X | 701.216 |
| 4.0X → 2.5X | 700.849 |
| 4.0X → 2.5X | 700.637 |
| 4.0X → 2.5X | 700.709 |
| 0.6X → 2.5X | 701.363 |
| 0.6X → 2.5X | 701.668 |
| 0.6X → 2.5X | 701.306 |
| 0.6X → 2.5X | 701.397 |

### 按方向分组统计

| 方向 | 次数 | Pixel 均值 | Pixel 范围 | 散布 |
|------|------|-----------|-----------|------|
| 0.6X → 2.5X (正向) | 5 | 701.317 | 700.852 ~ 701.668 | 0.816 |
| 4.0X → 2.5X (反向) | 4 | 700.853 | 700.637 ~ 701.216 | 0.579 |
| **全部** | **9** | **701.111** | **700.637 ~ 701.668** | **1.031** |

### 方向差异

正向均值 - 反向均值 = 701.317 - 700.853 = **0.464 pixel**

| 4.0X → 2.5X | 33169 | 33170 | +1 | 1 |

| 0.6X → 2.5X | 33169 | 33170 | +1 | 1 |
| 0.6X → 2.5X | 33169 | 33170 | +1 | 1 |
| 0.6X → 2.5X | 33169 | 33170 | +1 | 1 |

编码器散布：-1 ~ +1 counts（2 counts 范围）

## 分析

- 编码器定位精度 ±1 count，散布仅 2 counts
- 但 pixel 测量散布达 1.031 pixel，远大于 2 counts × 0.03 pixel/count = 0.06 pixel 的预期
- **方向差异 0.464 pixel**：正向到达（从 0.6X 来）比反向到达（从 4.0X 来）大约 0.5 pixel
- 即使编码器位置相同（都是 33170），从不同方向到达的 pixel 值也不同
- 说明存在**编码器无法感知的机械间隙**（齿轮/螺旋副间隙），编码器读数一样但光学实际位置不同

### 从不同方向到达 2.5X

| 路径 | Pixel 测量值 |

### 按方向分组统计

| 方向 | 次数 | Pixel 均值 | Pixel 范围 | 散布 |

编码器散布：-1 ~ +1 counts（2 counts 范围，与 DEADZONE=1 完全相同）

## DEADZONE=1 vs DEADZONE=3 对比

| 指标 | DEADZONE=1 | DEADZONE=3 |
|------|-----------|-----------|
| 编码器散布 | ±1 count | ±1 count |
| Pixel 全部散布 | 1.031 | 0.746 |
| Pixel 全部均值 | 701.111 | 701.559 |
| 正向均值 | 701.317 | 701.475 |
| 反向均值 | 700.853 | 701.665 |
| 方向差异 | +0.464 | -0.190 |

## 分析

- **编码器层面无差异**：两种 DEADZONE 的 count 散布完全相同（±1），说明 PID 惯性自然将电机带入目标 ±1 范围，DEADZONE=3 并未引入额外编码器误差
- **Pixel 散布主因不是 DEADZONE**：两轮整体均值偏移 0.45 pixel，方向差异甚至翻转，说明机械系统（温漂、摩擦、间隙）才是 pixel 散布的主要来源
- **DEADZONE=3 的 pixel 散布反而更小**（0.746 vs 1.031），可能是测试环境随机波动
