# 堵转与过流保护阈值标定报告

## 1. 测试环境

| 项目 | 参数 |
|------|------|
| 硬件平台 | ZLENS_DC 电动变倍镜头控制板 |
| MCU | STM32F103, 12-bit ADC (0~4095) |
| 电机驱动 | A4950 H 桥, VREF=2.5V (DAC 输出) |
| 电源电压 | 12V DC |
| 电流检测 | A4950 IPROPI 引脚 → ADC1 采样, IIR 滤波 (alpha=1/16) |
| PWM 频率 | TIM1, ARR=4266 |
| 编码器 | AS5311 磁编码器 (TIM4 正交解码) |
| 测试日期 | 2026-04-03 |

### 测试工况

- 手动堵住电机转子，分别在 30%、50%、80% 三个 PWM 占空比下采集电流 ADC 值
- 每个速度测 3 轮，每轮持续数秒
- 记录 IIR 滤波后的平均值 (filt avg) 和原始峰值 (raw max)

## 2. 测试结果

### 2.1 堵转电流实测（3 轮 × 3 速度）

| PWM (CCR) | 占空比 | 第 1 轮 avg | 第 2 轮 avg | 第 3 轮 avg | 平均 |
|-----------|--------|-------------|-------------|-------------|------|
| 1280      | 30%    | 1039        | 989         | 967         | 998  |
| 2133      | 50%    | 1770        | 1710        | 1685        | 1722 |
| 3413      | 80%    | 2742        | 2766        | 2733        | 2747 |

> 所有速度的 raw max 均达到 4089（ADC 满量程），说明瞬时电流已被 A4950 限流钳位，IIR 滤波后的稳态值才是有效参考。

### 2.2 正常运行参考值（历史数据, 15% PWM 回零）

| 工况 | filt 值 |
|------|---------|
| 空载 | ≈36 |
| 正常运动 | 平均 239, 峰值 942 |
| 堵转 (15% PWM) | ≈526~560 |

## 3. 阈值设计

### 3.1 系统运行速度范围

```
MAX_SPEED    = 1280 (30%)   — PID 输出上限, 远距离运动
中距离 cap   = 480  (11.2%) — error > 1000 时
近距离 cap   = 300  (7.0%)  — error ≤ 1000 时
MIN_SPEED    = 128  (3.0%)  — PID 输出下限
```

系统正常运行的最高占空比为 **30%**，堵转时 filt avg = **998**。

### 3.2 电流堵转阈值 (STALL_THRESHOLD)

| 参数 | 值 |
|------|-----|
| 阈值 | 1200 |
| 确认时间 | 1000ms (连续超阈值) |
| 消隐期 | 200ms (启动后) |

**取值依据：**
- 30% PWM 堵转 avg=998 < 1200，电流堵转检测在最大运行速度下**不会触发**
- 这是**设计意图**：电流堵转仅作为编码器堵转的备份保护
- 如果降到 800，正常运动峰值 942 太接近，存在误触发风险
- 不建议修改

### 3.3 编码器堵转检测

| 参数 | 值 |
|------|-----|
| 确认时间 | 500ms (编码器位置不变) |
| 独立于电流 | 纯机械/位置判断 |

**这是系统主力堵转检测方式**，不依赖电流阈值，直接检测电机是否卡住。

### 3.4 过流保护阈值 (OVERCURRENT_THRESHOLD)

| 参数 | 值 |
|------|-----|
| 阈值 | 3000 |
| 确认时间 | 50ms (连续超阈值) |

**取值依据：**
- 过流保护的目的是检测**硬件异常**（短路、驱动芯片故障），不是检测堵转
- 外部仪器实测：filt=3000 时系统总电流仅约 **800mA**，远未达到危险水平
- 80% PWM 堵转 filt avg=2747，接近但不超过 3000，不会误触发
- 正常运行最大 30% PWM，堵转 avg=998，filt 远低于 3000
- 阈值 3000 既能避免正常/堵转工况误触发，又能在真正短路等异常时保护硬件
- 保持 3000 不变

### 3.5 三级保护体系总结

```
 ┌─────────────────────────────────────────────────────────┐
 │                   三级电机保护体系                        │
 ├──────────────┬──────────┬──────────┬────────────────────┤
 │   保护层级    │  检测方式  │ 响应时间  │       作用         │
 ├──────────────┼──────────┼──────────┼────────────────────┤
 │ 1. 编码器堵转 │ 位置不变  │  500ms   │ 主力堵转检测       │
 │ 2. 电流堵转   │ filt>1200│ 1000ms   │ 备份 (30%不触发)   │
 │ 3. 过流保护   │ filt>3000│   50ms   │ 硬件异常紧急停止   │
 └──────────────┴──────────┴──────────┴────────────────────┘
```

## 4. 结论

- 编码器堵转检测是系统堵转保护的**核心机制**，可靠有效
- 电流堵转阈值 1200 作为备份保护，在当前 30% 最大占空比下不触发，属于正常设计
- 过流保护阈值保持 **3000**（对应实际总电流约 800mA，安全裕度充足）
- 如果未来修改 MAX_SPEED 或 VREF，所有电流相关阈值需要重新标定
